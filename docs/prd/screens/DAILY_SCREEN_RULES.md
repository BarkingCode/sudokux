# Daily Screen Rules

## Overview

Route: `app/(tabs)/daily.tsx`

The Daily screen presents a single global daily challenge that all users can compete on. Same puzzle for everyone, resetting at midnight local time.

---

## Layout Components

| Component | Description |
|-----------|-------------|
| Header | Date display + "DAILY" title |
| Challenge Card | Difficulty, grid type (always 9x9), play button |
| Streak Display | Current streak, best streak, milestones |
| Daily Calendar | Monthly completion history (green = completed) |
| Today's Leaderboard | Top 10 fastest times for today's puzzle |

---

## Ad Rules

### Banner Ads
| Location | Allowed |
|----------|---------|
| Daily Screen | ✅ **YES** |
| During Gameplay | ✅ **YES** |
| Completion Modal | ❌ **NO** |

### Interstitial Ads
- ❌ **NO interstitial ads** in Daily mode

### Rewarded Ads
- ❌ **NO rewarded ads** for Daily challenges
- Daily is ad-free to maintain competition integrity

### Ad-Free Subscribers
- No difference (Daily already has no ads)

---

## Game Rules

### Puzzle Generation
- Always **9x9** grid (no 6x6 option)
- Difficulty: Random from Easy, Medium, Hard only
  - ❌ Extreme, Insane, Inhuman excluded (accessibility)
- Same puzzle for all users worldwide
- Generated by server cron job at 12:00 UTC (for next day)

### Attempt Rules
- **One attempt per day** - cannot replay same day's puzzle
- Timer runs from first tap until completion
- Progress saved if user leaves mid-game
- Cannot restart once started

### Reset Timing
- Puzzle resets at **midnight local device time**
- Uses `getLocalDateString()` for date comparison

---

## Streak Rules

### Calculation
| Scenario | Result |
|----------|--------|
| Last completed = yesterday | Increment streak |
| Last completed = today | Keep current (no double-count) |
| Last completed = 2+ days ago | Reset to 1 |
| First ever completion | Start at 1 |

### Storage
- `user_stats.daily_streak` - current streak
- `user_stats.best_daily_streak` - all-time best

### Milestones
- 7-day streak - achievement + confetti
- 30-day streak - achievement + confetti

---

## Leaderboard Rules

### Display
- Top 10 fastest times for today's puzzle
- User's own rank highlighted if in top 100
- Shows username, time, rank

### Competition
- Global leaderboard (all users on same puzzle)
- Rank by `time_seconds` ascending
- Ties broken by `completed_at` (earlier wins)

---

## Calendar Component

### Visual States
| State | Color |
|-------|-------|
| Completed | `colors.success` (green) |
| Missed (past) | `colors.highlight` (gray) |
| Today | Thick border |
| Future | Faded/disabled |

### Features
- Month navigation (< > buttons)
- "X completed" count per month
- Legend at bottom
- Only visible to logged-in users

---

## UI/UX Highlights

### Tab Notification
- Red dot on Daily tab when today's puzzle incomplete
- Clears when user completes daily

### Completion Flow
1. User completes puzzle
2. DailyCompletionModal shows:
   - Time, mistakes, hints used
   - User's leaderboard rank
   - Top 10 times
3. "View Leaderboard" or "Back to Daily" options

### Offline Behavior
- Cache today's puzzle on app open
- If server unavailable: "Unavailable" message + "Try Again"
- No client-side fallback (ensures fair competition)

---

## Push Notifications

### Daily Reminder
- Sent at user's preferred time (default: 9:00 AM local)
- Message: "Your daily Sudoku challenge is waiting!"
- Only sent if puzzle incomplete
- Cron checks every 15 minutes
